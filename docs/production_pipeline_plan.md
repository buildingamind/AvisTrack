# Production Tracking Pipeline — Implementation Plan

## Goal

Run all videos in `E:\Wave2\00_raw_videos\` using the `top9_interp` model.
Output one parquet file per video (1:1 correspondence by stem name) to `E:\Wave2\05_Tracking_Output\`.
Each parquet includes a `unix_time` column. CCM consumes these parquets directly for analysis.

---

## Architecture Rule

**Backends handle inference and interpolation only. All I/O lives in `run_batch.py`.**

This keeps the pipeline model-agnostic — swapping YOLO for another backend later
requires no changes to output logic.

```
offline.py:   flush_interpolation(rows) → DataFrame   (no file writing)
run_batch.py: DataFrame → append unix_time → write parquet
```

## Files to Change

### 1. `avistrack/backends/yolo/offline.py`

Change `flush_interpolation()` to return a DataFrame instead of writing to disk.

```python
# Before
def flush_interpolation(self, rows, output_path) -> None:
    ...
    df.to_csv(output_path, index=False, header=False)

# After
def flush_interpolation(self, rows) -> pd.DataFrame:
    ...
    return df
```

Returned DataFrame columns: `frame (int32), id (int16), x/y/w/h (float32), conf (float32), class (int8)`

---

### 2. `cli/run_batch.py`

Replace the existing `flush_interpolation` call in `_process_video()`:

```python
# 1. interpolate → get DataFrame
df = tracker.flush_interpolation(rows)

# 2. append unix_time
time_lookup = _load_time_lookup(cfg, video_path.name)
df["unix_time"] = (
    df["frame"].map(time_lookup.frame_to_unix).astype("float64")
    if time_lookup else float("nan")
)

# 3. write parquet or mot
output_format = getattr(cfg.output, "format", "mot")
if output_format == "parquet":
    out = Path(cfg.output.dir) / (video_path.stem + ".parquet")
    df.to_parquet(out, engine="pyarrow", compression="zstd", index=False)
else:
    out = Path(cfg.output.dir) / (video_path.stem + ".txt")
    df.drop(columns=["unix_time"], errors="ignore").to_csv(out, index=False, header=False)
```

Add helper function `_load_time_lookup(cfg, video_name) -> TimeLookup | None`:
- Reads `cfg.drive.time_calibration`
- Looks up `video_name` key in the JSON
- Returns `TimeLookup.from_calibration(...)` or `None` if key is missing
- Missing key → `unix_time` column is all NaN; batch continues normally (checkpoint unaffected)

`time_calibration.json` key format (already generated by `tools/calibrate_time.py`):
```json
"Wave2_CollectiveDanger_Day0_CAM1.mkv": {
    "fps_nominal": 30.0,
    "samples": [{"frame": 0, "unix": 1750293992.0}, ...]
}
```
`TimeLookup.from_calibration()` already supports this format — just call it.

**Final parquet schema:**

| Column | Type | Notes |
|--------|------|-------|
| `frame` | int32 | frame index |
| `id` | int16 | track ID (1–9) |
| `x` | float32 | bbox left-x, perspective-corrected |
| `y` | float32 | bbox top-y |
| `w` | float32 | bbox width |
| `h` | float32 | bbox height |
| `conf` | float32 | detection confidence |
| `class` | int8 | class label (0) |
| `unix_time` | float64 | Unix timestamp (seconds) |
